Aula_9 - Introdução a position

Position Static: Como o nome sugere, define o posição quando declarado como estático.

Position fixed: Esse postion quando define deixa como fixo o conteúdo em relação a tela do navegador.

Position Absolute: Irá ficar posicionado fixo no documento em relação a um ancestral que esteja posicionado mas não estatico, mas caso não encotre nenhum ancestral posicionado ele ficará posicionado em relação ao documento e não a janela do browser.

Position Sticky: Funciona como um limitador, ou seja, ao definir esse position como sticky no topo, o scroll ao ir passando (descendo) e chegar ao position stucky o conteudo irá ficar grudado do topo sempre.

Aula_10 - Introdução ao FlexBox

O flexbox: Flexing box module, é modulo de caixa flexivel. Temos o box model que faz com que nossos elemento fiquem em "caixas", mas com o flexbox podemos esticar essas caixas, organizar de acordo com a sua vontade, fazer o alinhamento que fique melhor visualmente. Para definir se estaremos usando colunas ou linhas, usamos a propriedade flex-direction para definir qual será o eixo principal (y, x).

Aula_11 - Valores Globais para Propriedades

São valores que podemos usar em todas propriedades css, mas como toda regra tem exceção, existem alguns valores que não podem ser usados em algumas propriedades.

Initial: Como o nome pode sugerir (pelo menos para esse autor), irá redefinir a propriedade escolhida para o valor inicial da mesma. Resumindo retornar ao valor padrão.

Inherit: Esse valor irá definir os valores de um pai para seu herdeiros, pois, a maioria das propriedades não herdao os valores definidos nos pais.

Unset: Esse valor irá desabilitar (desligar) a propriedade padrão.

Revert: Exclusivo do safari, esse valor irá ter a mesma função do initial.

Aula_12 - Indrodução ao Grid Layout

Grid Layout um grande recurso do CSS3, foi criado para a cosntrução de layouts, especificamnete para esse fim. A sua principal função assim como do flexbox, que também foi criado com finalidade para layouts, o Grid Layout permite manipular os eixos Y e X para alinhamento, diferente do flexbox.

Obs: O flexbox é sim muito util para criação dos layouts, o uso de cada uma das técnicas irá depender da necessidade e demanda

Como mencionado o Grid Layout irá permiir um direcionamento bi-direcional, mas é sempre bom enfativar que com grandes poderes vem grandes responsabilidades, o grid layout acaba sendo um pouco mais complexo em comparação ao flexbox por causa dos enormes recurso que existe.

Porque usar o grid? Diferente doque posso ficar subentendo, grid layout pode sim ser usado com outros métodos tal comoo flexbox.

GridLayout tem alguns elementos particulares, tais como grid line que são todas as linhas do grid, mas não as linhas da borda e sim as internas. E tem o grid cells que são as celulas do elemento grid. O grid track é o conjunto de celulas agrupadas, tanto vertical quanto horizontal mas apenas um sentdo por vez. O grid area é diferente do grid track pois agrupa uma area conjunta vertical e horizontal.

No grid layout existe o Gutters que seria os espaços entre as células.

Um dos tipos seris o combinador descedente, que seria o nome de dois seletores separados apenas pelo espaço.

Combinador de irmão adjancente é aquele que vem com o sinal de adição (+).

Combinador de irmão geral é aquele que vem com o sinal til (~).
